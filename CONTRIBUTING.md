# Contributing to Docryn

Thank you for your interest in contributing to Docryn.

## Our Philosophy

Before writing code, we design.

Every significant feature starts with:

- RFC
- ADR (when required)
- Specification
- Conformance Tests
- Implementation

## Development Workflow

1. Open an Issue
2. Discuss the proposal
3. Write or update the RFC
4. Submit a Pull Request
5. Pass all tests
6. Merge

## Coding Standards

- Keep code simple.
- Write tests.
- Update documentation.
- Preserve backward compatibility whenever possible.

## Questions

Open a GitHub Discussion before implementing major changes.
